### 💡 設計メモ（2026-02-22）

#### シフト希望の種類（将来的な拡張）
```
タイプ1: 確定希望 (fixed)
  → そのままインポート（濃い色）
  → まずはこれだけ実装

タイプ2: 条件付き希望 (flexible) ← 将来実装
  → 管理者が時間調整（中間色）
  → 例: "12:00-20:00の間で4時間"
  → 備考欄にテキストで記載

タイプ3: おまかせ希望 (any) ← 将来実装
  → 管理者が日付と時間を決定（薄い色）
  → 例: "2/5-2/10の間で適当に"
  → 備考欄にテキストで記載
```

#### UI設計の方針
- **編集画面**: スプレッドシート形式（案A）
- **閲覧画面**: カレンダービュー（スタッフ用など）
- **インポート済みでないシフト希望の表示**: 要検討

#### データ構造（CSV）
```csv
staff_id,day,start_time,end_time,request_type,note,read_status,created_at,updated_at
S001,1,12:00,20:00,fixed,,unread,2025-02-01 10:00,2025-02-01 10:00
S002,2,12:00,20:00,fixed,午後希望,read,2025-02-01 11:00,2025-02-01 11:00
```
